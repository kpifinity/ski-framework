# v3.1.0-beta.1 — Beta hardening: eval loop, L3 6/6, benchmarks, vLLM, Helm, /metrics

**Released:** 2026-06-13 · **Pre-release** (first beta of the v3.1 line)

> ⚠️ This is a pre-release. The API and envelope format are stable for
> testing; production deployments should track the v3.0.x stable line
> until v3.1.0 GA.

## Install

```bash
pip install --pre ski-sdk ski-kg-extractor ski-kg-validator ski-audit-ledger ski-model-deploy ski-schemas
docker pull ghcr.io/kpifinity/ski-model:3.1.0-beta.1
```

## Highlights

### No unverified CLEARs — ever
The taxonomy guard closes the deepest gap the eval suite found: a model that
reasons "nothing maps" can no longer emit CLEAR with zero citations and zero
assertions. CLEAR now requires verified satisfaction — assertion-less CLEARs
are deterministically remapped to NULL_UNMAPPED (no citations) or DISCRETIONARY
(citations present) and recorded in `envelope.notes`. Across every eval run to
date, **breaches silently CLEARed = 0**.

### Sovereignty conformance: 6/6 runnable
The air-gapped boot rig (`conformance/sovereignty/test_air_gapped.py`) completes
the L3 suite. It boots the audit ledger with `--network=none`, joins the SKI
Model to the same loopback-only namespace, replays a five-measurement
signed-KG workload, and verifies the hash-chained ledger from inside the gap.
Every L3 check now gates CI. Conformance suite revision: **0.5.0**, 77 tests.

### SKI Evals — verdict accuracy, measured not asserted
The eval suite (`evals/`) ships a 50-case human-labeled golden dataset evaluated
through the real production path. Published metrics: verdict accuracy, FLAG
recall/precision, NULL_UNMAPPED recall, assertion correctness, LLM↔verifier
agreement rate. Methodology: `docs/evals.md`.

### Framework overhead: 0.36 ms p99
The benchmark suite isolates framework overhead from LLM inference. Reference:
**p50 0.10 ms / p95 0.16 ms / p99 0.36 ms** on 2 vCPUs — ~250× inside the
100 ms budget. CI gates on p99 ≤ 100 ms. Methodology: `docs/benchmarks.md`.

### Observation grounding — fabricated observations no longer pass
Every field of a formalizable assertion is now verified against a source of
truth the model cannot invent. `assertion.metric` must exist in the measurement
record, `assertion.observed` must match the recorded value, `assertion.value`
must match the cited obligation — any mismatch is LLM_CONTRADICTION → DISCRETIONARY.

### Production GPU inference: vLLM backend
`SKI_V3_LLM_BACKEND=vllm` routes to OpenAI-compatible `/v1/chat/completions`
with vLLM's decoder-level `guided_json` grammar enforcement (token masking —
stronger than post-hoc validation).

### Kubernetes: Helm chart with constraints enforced at render
`deploy/helm/ski` — render fails on `replicas`/`replicaCount` (single writer),
fails without an operator-supplied Secret, fails without a signed-KG ConfigMap.
NetworkPolicy enforces the sovereign boundary; `networkPolicy.airgap=true`
drops DNS. Default image: `ghcr.io/kpifinity/ski-model` (no manual `--set` required).

### Wire contract: one definition (`ski-schemas`)
`tools/ski-schemas` (PyPI: `ski-schemas` 0.1.0) — verdict envelope, signed
transcript, and measurement record live in one place. `ski_model.v3.envelope`
and `.transcript` re-export from it; drift is structurally impossible.

### `/metrics` — observability contract, implemented
Every series the Prometheus alert rules fire on is now exported:
`ski_agreement_rate`, `ski_kg_signature_verified`, `ski_ledger_sequence_gaps_total`,
`ski_verdicts_total{verdict=}`, `ski_evaluation_duration_seconds`,
`ski_model_last_telemetry_timestamp`, `ski_runtime_info`. A regression test
pins the contract so it cannot silently drift.

### Validator ↔ verifier predicate parity
The Symbolic Verifier now checks every mechanically checkable §3.3 obligation
type (`must_be_below`, `must_be_above`, `must_be_one_of`, `must_not_be_one_of`).
A parity test pins both directions.

### EU AI Act crosswalk
`docs/crosswalks/eu-ai-act.md` — article-level mapping of SKI controls to
Regulation (EU) 2024/1689 ahead of the 2 August 2026 applicability date.
Informative, not legal advice.

### 5-minute demo
`docker-compose.demo.yml` — full v3 pipeline under FakeLLM. No multi-GB model
pull between a new user and their first verdict envelope.

## What's new since alpha.2

Everything above. The alpha.2 release shipped: ski-sdk v0.1.0, 4/6 L3 checks,
first PyPI publication, and fully gating CI. Beta.1 completes the remaining
engineering scope: the 6th L3 check, three eval-loop runtime fixes, ski-schemas,
/metrics, predicate parity, vLLM, Helm, benchmarks, EU AI Act crosswalk, and
working quickstart.

## Upgrade from alpha.2

No wire format changes. Import shims are in place.

```bash
pip install --pre --upgrade ski-sdk ski-kg-extractor ski-kg-validator \
    ski-audit-ledger ski-model-deploy ski-schemas
```

## Known limitations

- Real-model accuracy numbers still maturing; follow `docs/evals.md`.
- vLLM backend requires an external GPU endpoint.
- `ski-schemas` 0.1.0 independently versioned; may reach 1.0.0 before the framework.
